## Get started with a template

The easiest way to get started is to use our **Vite + JavaScript** template.  
It comes preconfigured with Panda CSS, Ark UI, and Fidely UI and theme support.

```bash
npx degit https://github.com/fidely-ui/vite-js-template
```

---

## Manual Installation

If you prefer setting things up yourself, follow the steps below.

## 1. install Panda CSS

```bash
npm install -D @pandacss/dev
npx panda init --postcss
```

## 2. install Ark UI

Ark UI provides the headless, accessible primitives used by Fidely UI.

```bash
npm install @ark-ui/react
```

## 3. install Fidely UI

```bash
npm install @fidely-ui/react @fidely-ui/panda-preset
```

## 4. Update package.json scripts

Open your **package.json** file and add the **prepare** script:

```json
{
  "scripts": {
+    "prepare": "panda codegen",
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

The **prepare** script ensures Panda CSS generates styles automatically when dependencies are installed.

## 5. Configure path aliases (JavaScript)

Since this is a JavaScript project, create a **jsconfig.json** file at the root

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "skipLibCheck": true,
    "baseUrl": ".",
    "paths": {
      "styled-system/*": ["styled-system/*"]
    }
  }
}
```

This allows Panda’s generated **styled-system** files to resolve correctly.

## 5.5 Configure Vite path aliases

Vite does not automatically read **jsconfig.json** or **tsconfig.json** path
aliases at runtime.

You must explicitly configure aliases in **vite.config.js**.

### Install Node types

Install Node types to avoid errors when importing Node built-ins like path

```bash
npm install -D @types/node
```

Update **vite.config.js**

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'node:path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      'styled-system': path.resolve(__dirname, 'styled-system'),
    },
  },
})
```

### Why we don’t use **vite-tsconfig-paths**

CSS imports are not supported.
Due to a Vite limitation, CSS files (and CSS dialects) cannot be resolved with this plugin.

## 6. Configure Panda CSS

Update **panda.config.ts**

```ts
import { defineConfig } from '@pandacss/dev'
import { fidelyPreset } from '@fidely-ui/panda-preset'

export default defineConfig({
  presets: [fidelyPreset()],

  // Enable CSS reset
  preflight: true,

  // Files Panda should scan for styles
  include: [
    './src/**/*.{js,jsx,ts,tsx}',
    './components/**/*.{js,jsx,ts,tsx}',
    './node_modules/@fidely-ui/react/**/*.{js,jsx,ts,tsx}',
  ],

  exclude: [],

  // Theme customization
  theme: {
    extend: {},
  },

  // Generate static CSS for all recipes
  staticCss: {
    recipes: '*',
  },

  // Output directory for generated styles
  outdir: 'styled-system',

  jsxFramework: 'react',

  // Remove Panda default 50 / 950 color steps
  plugins: [
    {
      name: 'panda-headless-colors',
      hooks: {
        'preset:resolved': ({ utils, preset, name }) => {
          if (name === '@pandacss/preset-panda') {
            return utils.omit(preset, [
              'theme.tokens.colors',
              'theme.semanticTokens.colors',
            ])
          }
          return preset
        },
      },
    },
  ],
})
```

After updating the config, generate the styled system:

```bash
npx panda codegen
```

## 7. Configure the entry CSS with layers

Open **src/App.css** and replace its contents with:

```css
@layer reset, base, tokens, recipes, utilities;
```

> You can safely remove unused CSS files if present and delete **index.css**.

## 8. Import global CSS in the app entry

Update src/main.tsx:

```jsx
import { StrictMode } from "react"
import { createRoot } from "react-dom/client"

import "./App.css"
import App from "./App"

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

## Enjoy ✨

You can now start using Fidely UI components:

```jsx
import { Button } from '@fidely-ui/react'

const Demo = () => {
  return <Button>Click me</Button>
}
```
